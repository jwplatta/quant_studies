## Change Tested

Tested experiment 7 from the repair matrix: when a live 7DTE SPXW iron condor
reaches short-leg delta `>= 0.25` or breaches a short strike while in the `1-2`
DTE bucket before `12:00` ET, the strategy closes the position and re-enters a
fresh iron condor on the same expiry.

## Method

Ran the standalone project `qc/spxw_7dte_exp07_recenter_late_am` in
QuantConnect cloud and parsed the saved results payload from
`research_log/results/spxw_7dte_exp07_recent_late_am.json`. The tested code
version is recorded as commit `3d61391` (`fix exp07 greek access`), which
matches the saved run name `exp07_2026-04-06_greek_fix`. The project used
minute SPX and SPXW data, baseline entry constants, and the same
`IronCondorFinder` as the no-constraints baseline.

## Interpretation

This result is worse than the late-cycle roll-forward test in `exp06` on both
headline PnL and drawdown, while still requiring higher turnover. Same-expiry
recentering this late in the trade did not rescue the position; instead it
appears to have added churn without restoring enough edge or reducing portfolio
stress. On these results, late-cycle recentering should be treated as rejected.
